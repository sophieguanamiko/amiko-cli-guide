CLI 命令分两类

🛠

系统命令 system

CLI 自带的固定指令，语法固定、输出稳定。
比如 amiko login、amiko wallet balance

🎨

用户技能 skill

你或社区创建的能力扩展包。
比如 amiko music composer、amiko daily report

🆚 Amiko CLI vs Lark CLI

两个 CLI 我们都做了，区别在这

维度Lark CLIAmiko CLI
定位组织协作工具个人 AI 操作系统
管理对象群、文档、日历、审批AI 分身、好友、钱包、记忆、创作
典型场景「发通知到团队群」「让我的 AI 分身每天给我发简报」
AI 角色工具（处理文档/消息）分身（替你做决策、执行、社交）
🏢

Lark CLI

管团队的协同事务 — 建群、写文档、发消息、排日历。所有「和一群人一起做事」的场景。

🌙

Amiko CLI

管自己的数字生活 — 加好友、审群聊、做音乐、管钱包、建分身。所有「一个人掌控自己」的场景。

💡 Lark 管「事」，Amiko 管「我」。不是一个赛道，可以同时用。

⚡ 为什么要用 CLI 而不是 App？

三个比你拍脑袋想到的更实在的理由

⏱

效率：秒杀鼠标

「查余额」→ 1 句话 vs 打开 App → 点钱包 → 等加载 → 翻页面。
CLI 把分钟级操作变成秒级。

🔁

自动化：设一次就不用管

「每天早上 9 点给我发 AI 行业简报」—— 说一次，你的 AI 每天自动用 CLI 搜索、整理、定时发给你，不需要闹钟不需要手动触发。

🧩

可组合：多个能力串联

「在 X 上搜热门话题 → 写一篇分析 → 生成配图 → 发到我的群」—— CLI 让 AI 把搜索+创作+分发串成一条自动化流水线。
